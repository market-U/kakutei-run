# design: issue-19-enlarge-enemy

## Context

現在の敵キャラクターは `FrameSize.ENEMY = { width: 64, height: 96 }` のスプライトシートを使用し、スケール設定なし（等倍）で表示されている。初期距離は `INITIAL_ENEMY_DISTANCE = GAME_W / 2 - 20 = 460px` で、プレーヤー（画面中央 x=480）から 460px 左に配置され、幅 64px スプライトの右端約 52px が画面左端に見える状態である。

## Goals / Non-Goals

**Goals:**

- 敵の表示サイズを現在の約1.5倍に拡大する（フレームサイズ 64×96px → 96×144px）
- 拡大後もチラ見えによる「迫ってくる感」の演出を維持する
- 当たり判定が新しいスプライトサイズに自然に追従する

**Non-Goals:**

- ゲームプレイロジック（速度・被ダメージ回数・ゴール判定）の変更
- プレーヤーや魔女など他キャラクターのスケール変更
- 初期距離の値そのものを変える（`INITIAL_ENEMY_DISTANCE` は変更しない）

## Decisions

### アセット差し替えによる拡大（setScale を使わない）

**採用**: `FrameSize.ENEMY` を `{ width: 96, height: 144 }` に更新し、同サイズのスプライトシートファイルを差し替える。

**理由**: `setScale(1.5)` は実装は簡単だが、既存素材を拡大するため将来的に高解像度素材に差し替えた際に対応が二段階になる。ユーザーが新サイズのアセットを用意できるため、最初から正確な解像度で管理するほうが一貫性が高い。

**代替案**: `this.setScale(1.5)` を Enemy コンストラクタに追加する方法も検討したが、不採用とした。

---

### 初期距離の維持

**採用**: `INITIAL_ENEMY_DISTANCE = GAME_W / 2 - 20 = 460` を変更しない。

**理由**: 新スプライト半幅が 48px になることで、チラ見え量は約 52px → 約 68px に増加する。これは「より見えやすくなる」という本変更の意図と一致しており、追加の距離調整は不要である。

---

### 当たり判定の扱い

**採用**: `Enemy.getHitBounds()` はすでに `this.displayWidth` / `this.displayHeight` を参照しているため、スプライトフレームサイズ変更後も自動的に新サイズに追従する。コード変更不要。

## Risks / Trade-offs

- **スプライトファイルが用意できるまでプレースホルダーが旧サイズのまま**  
  → `scripts/generate-placeholders.mjs` の ENEMY フレームサイズも更新し、差し替え前から動作確認できるようにする
- **フレームサイズ変更後に既存スプライトシートが残っていると表示が崩れる**  
  → `public/assets/sprites/enemy_run.png` を新サイズで上書きするまで旧ファイルがあるとズレが生じる。アセット差し替えとコード変更は同一コミットで行うこと
