# Proposal: initial-game

## Why

確定申告の締切という日本人に身近なテーマを使い、誰でもURLからすぐに遊べるフロントエンド完結型のWebゲームを新規に作成する。
バックエンドもアカウントも不要で気軽に遊べる体験を提供し、SNSシェアによる拡散を狙う。

## What Changes

新規プロジェクトとして、以下の構成要素を すべてゼロから実装する。

- Phaser 3 + TypeScript + Vite によるゲームアプリケーションの土台構築
- チャージジャンプを中心とした操作システム
- 横スクロールゲームループ（視差スクロール・ランダムマップ生成）
- 障害物・アイテム・敵キャラクターシステム
- レシート回収率スコアシステム
- タイトル・ゲーム本体・リザルトの3シーン構成
- スプライトシート・音声のアセット差替え可能な管理構成
- 難易度設定（Easy / Normal / Hard）
- Azure Static Web Apps へのデプロイ構成
- ロジック層のユニットテスト（Vitest）

## Capabilities

### New Capabilities

- `player-mechanics`: チャージジャンプ（長押しで最大500msまで溜め、離したら発動）、プレーヤーの状態管理（通常走行・腰痛スロー・転倒・ゴール）
- `game-world`: 視差スクロール（遠景・近景・地面の多層）、ランダムマップ生成（石ころ・レシート・魔女の配置）
- `obstacles`: 石ころ（地面設置、接触でゲームオーバー）、魔女（空中浮遊、接触で腰痛スロー・3回でゲームオーバー条件）
- `receipt-scoring`: レシート（地面・空中配置、接触で回収）、ゴール時の回収率をスコアとして算出・表示
- `enemy`: 敵キャラクターの追跡ロジック（画面左端にチラ見え、魔女被弾による間隔縮小、追い付かれたらゲームオーバー）
- `scenes`: タイトルシーン（難易度選択含む）・ゲームシーン・リザルトシーン（SNSシェア含む）の3画面構成
- `difficulty`: Easy / Normal / Hard の難易度パラメータ管理（レシート枚数・障害物密度・スクロール速度など）
- `assets`: スプライトシート・BGM・SEのアセット管理構成（差替え可能な構成、プレースホルダー含む）

### Modified Capabilities

（新規プロジェクトのため既存スペックなし）

## Impact

- 新規リポジトリ（GitHub）
- 新規フロントエンドアプリ（Phaser 3 / TypeScript / Vite）
- デプロイ先: Azure Static Web Apps
- 外部依存: `phaser`, `vite`, `typescript`, `vitest`
- グラフィックアセット: オーナーが別途用意（初期はプレースホルダー画像で代替）
- 音声アセット: 優先度低、後から差し替え可能な構成にしておく
