# Design: state-based-sprite-animation

## Context

現在の `Player.ts` と `Enemy.ts` はスプライトアニメーションを「走行」「腰痛走行」「転倒」「ゴール」の4種類で管理している。しかし以下の問題がある。

- `Player.update()` にはジャンプ中のアニメーション制御が未実装（コメントのみ）であり、ジャンプ中も走行アニメーションが流れ続ける
- 税務署接触後（`triggerGoal()`）にゴールアニメーションは再生されるが、ジャンプチャージ時のY方向スケール縮小は未実装
- 敵キャラクター（`Enemy.ts`）はプレーヤーに追い付いた後も走行アニメーションが継続している
- ジャンプチャージ中のスプライトY縮小演出が未実装

## Goals / Non-Goals

**Goals:**

- ジャンプ中に `Player` のアニメーションを停止する（離地時に停止、着地時に再開）
- 税務署接触後に `Player` のアニメーションを停止する
- `Enemy` がプレーヤーに接触した後にアニメーションを停止する
- 腰痛スロー中に専用スプライト `player_back_pain` のアニメーションを正しく再生する
- チャージ量 (0〜1) に連動してプレーヤースプライトを Y 方向に縮小する（下端起点）

**Non-Goals:**

- 新規スプライトアセットのデザイン・画像制作（グラフィック制作は別タスク）
- プレーヤー以外のオブジェクトのチャージ演出
- アニメーション状態をネットワーク同期する機能

## Decisions

### 1. ジャンプ中のアニメーション停止

`releaseJump()` 内でジャンプを発動する際に `this.anims.stop()` を呼び出してフレームを固定する。着地時の `onLanded()` で `playAnim()` を呼び出して再開する。

**代替案**: 専用の `jump` アニメーション状態を追加する
→ スプライトアセットが不要なため `stop()` で固定するシンプルな方法を採用。

### 2. 税務署接触後のアニメーション停止

`triggerGoal()` 内でゴールアニメーション再生後、または即座に `anims.stop()` を呼ぶことは既存仕様（ゴールアニメーション再生）と競合するため、`player_goal` アニメーションは `repeat: 0`（1回再生で停止）に変更し、アニメーション完了時に自動で最終フレームに固定する。

**代替案**: `on('animationcomplete', ...)` でコールバック → `repeat: 0` の方が実装がシンプルであるため採用。

### 3. Enemy 接触後のアニメーション停止

`Enemy.stopChasing()` メソッド内で `this.anims.stop()` を呼び出す。このメソッドは衝突後に `CollisionManager` から呼ばれるため、追加フックなしで対応できる。

### 4. チャージ中のY方向スケール縮小

`Player.update()` 内でチャージ量 `getChargeAmount()` を参照し、チャージ中のみ `scaleY` を動的に変更する。縮小は下端起点のため `origin` は `(0.5, 1)` が既に設定されており追加変更不要。

- チャージ 0%: `scaleY = 1.0`
- チャージ 100%: `scaleY = 0.7`（30%縮小）
- 計算式: `scaleY = 1.0 - 0.3 * chargeAmount`
- チャージ非中（離地 or チャージなし）の場合は `scaleY = 1.0` にリセット

**代替案**: Tweenアニメーションで実装する → 毎フレーム量に連動させる仕様のため `update()` 内での直接制御を採用。

### 5. 腰痛スロー専用アニメーション

`activateBackPain()` でのアニメーション切り替えは既に実装済み。ただし将来的に `PLAYER_BACK_PAIN` スプライトアセットがプレースホルダーの場合、本実装では正式アセットに差し替えることを前提とする。アセット制作は本chageのスコープ外。

## Risks / Trade-offs

- **[リスク] `scaleY` の直接変更が物理ボディに影響する** → Phaser の `Arcade.Body` はスプライトスケール変更に自動追従しないため、見た目のみ圧縮され当たり判定は変わらない。意図通りのため問題なし。
- **[リスク] `repeat: 0` でゴールアニメーションが1回しか再生されない** → ゴール後はリザルトシーンへ遷移するため実用上問題なし。
- **[トレードオフ] チャージ非中での `scaleY = 1.0` リセット処理が毎フレーム走る** → 計算コストは無視できる程度で問題なし。
