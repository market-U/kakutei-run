# Spec: player-mechanics

## MODIFIED Requirements

### Requirement: プレーヤー状態管理

プレーヤーキャラクターは以下の状態を持ち、状態に応じてアニメーションが切り替わる。

#### Scenario: 通常走行状態

- **WHEN** プレーヤーが地面上にいて障害物に接触していない
- **THEN** 走行アニメーションが再生される

#### Scenario: 腰痛スロー状態への遷移

- **WHEN** プレーヤーが魔女に接触する
- **THEN** スクロール速度が `gameConfig.ts` の `witchSpeedReduction`（速度低下率）だけ低下し、腰痛専用スプライトアニメーションが再生される。敵キャラの速度は変化せず、速度差によって敵との距離が縮まり続け、`witchDistanceFraction` 分の距離が詰まった時点で腰痛スローが解除され通常状態に戻る

#### Scenario: 転倒状態への遷移

- **WHEN** プレーヤーが石ころに接触する
- **THEN** 転倒アニメーションが再生され、スクロールが停止し、ゲームオーバーシーケンスに移行する

#### Scenario: ゴール状態への遷移

- **WHEN** プレーヤーが税務署（ゴール）に到達する
- **THEN** ゴールアニメーションが1回再生（ループなし）された後、最終フレームに固定され、クリアシーケンスに移行する

#### Scenario: ジャンプ中のアニメーション停止

- **WHEN** プレーヤーが地面を離れ空中に移行する
- **THEN** アニメーション再生が停止し、スプライトが現在のフレームに固定される

#### Scenario: 着地後のアニメーション再開

- **WHEN** プレーヤーが地面に着地する
- **THEN** 腰痛スロー状態であれば腰痛専用アニメーションが、そうでなければ走行アニメーションが再開される

#### Scenario: 敵キャラクター接触後のアニメーション停止

- **WHEN** 敵キャラクターがプレーヤーに到達し `enemyReached` イベントが発火する
- **THEN** プレーヤーのアニメーション再生が停止し、スプライトが現在のフレームに固定される

## ADDED Requirements

### Requirement: チャージ中の Y 方向スケール縮小

プレーヤーがジャンプチャージ中、チャージ量に連動してスプライトを Y 方向に縮小する。縮小はスプライト下端を起点に行う。

#### Scenario: チャージ開始時のスケール変化

- **WHEN** プレーヤーがジャンプチャージを開始する
- **THEN** チャージ量が 0% の時点では `scaleY = 1.0` のままである

#### Scenario: チャージ最大時のスケール

- **WHEN** プレーヤーのチャージ量が 100% に達する
- **THEN** スプライトの `scaleY` は 0.7 となり（下端起点で 30% 縮小）、それ以上は縮小しない

#### Scenario: チャージ量に連動したスケール

- **WHEN** プレーヤーのチャージ量が 0〜100% の間にある
- **THEN** `scaleY = 1.0 - 0.3 * chargeAmount` の計算式に従ってスケールが設定される

#### Scenario: ジャンプ発動後のスケールリセット

- **WHEN** プレーヤーがジャンプを発動する（チャージをリリースする）
- **THEN** `scaleY` が 1.0 にリセットされる
