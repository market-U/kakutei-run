# Proposal: issue-3-title-result-ui-redesign

## Why

タイトル画面・リザルト画面が横画面表示時に要素がはみ出す問題があり、スマートフォン・PC 問わず快適に操作できない。また、スタートボタンが難易度選択と分離していてタップ数が多い UX の改善と、Tailwind CSS + DaisyUI によるデザインの向上・保守性の改善が求められている。

## What Changes

- **Tailwind CSS + DaisyUI を導入**し、インラインスタイルをユーティリティクラスへ移行する
- **タイトルロゴのプレースホルダー画像を配置**する（後からアセット差し替えのみで対応できる構造にする）
- **スタートボタンを廃止**し、難易度ボタン1タップで即ゲーム開始できるようにする（リザルト画面のリトライボタンと同じ挙動）
- **レスポンシブレイアウト対応**：
  - 縦画面：従来どおりの縦並びレイアウト
  - 横画面：ロゴ・タイトルを左カラム、難易度ボタン等を右カラムの2カラムレイアウトに切り替える（タイトル画面・リザルト画面ともに同様）
- **アプリバージョン表記**を追加する（`package.json` のバージョンを Vite 経由で埋め込む）
- タイトル画面・リザルト画面の**難易度ボタンを共通コンポーネント化**する
- **リザルト画面も同様にレイアウト修正**を行う

## Capabilities

### New Capabilities

### Modified Capabilities

- `html-ui`: タイトル画面のゲーム開始フロー変更（スタートボタン廃止・難易度ボタン1タップ即開始）、Tailwind CSS + DaisyUI 導入によるスタイル刷新、縦横レスポンシブレイアウト対応（横画面は2カラム）、タイトルロゴ画像・バージョン表記の追加、難易度ボタンの共通コンポーネント化（タイトル画面・リザルト画面で共通化）

## Impact

- `index.html`: インラインスタイルを Tailwind クラスへ置き換え。構造も一部変更（ロゴ要素・バージョン表記の追加、スタートボタン削除、横画面2カラム用ラッパー追加）
- `src/ui/TitleUI.ts`: 難易度ボタンのクリックで即 `kakutei:startGame` を発火するよう変更
- `src/ui/ResultUI.ts`: 難易度ボタンを共通コンポーネントとして切り出し
- `src/ui/DifficultyButtons.ts`（新規）: タイトル・リザルトで共通利用する難易度ボタンコンポーネント
- `package.json` / `vite.config.ts`: バージョン埋め込み設定
- 新規依存: `tailwindcss`, `@tailwindcss/vite`, `daisyui`
