## Context

現在、ジャンプチャージ中のプレーヤースプライトはY方向（縦）のみを縮小している（最大20%縮小、scaleY = 0.8）。X方向のスケールは常に1.0のまま。

チャージ量に連動したX方向拡大を追加することで、スクワット時に横方向へ体が広がる見た目を実現し、「力を溜めている」感覚を高める。

## Goals / Non-Goals

**Goals:**
- チャージ量に比例してX方向を最大1.2倍まで拡大する
- Y方向縮小と同一のチャージ量計算を共用して実装コストを最小化する
- 既存のリセット処理（`setScale(1.0, 1.0)`）はそのまま流用する

**Non-Goals:**
- X方向の当たり判定変更は行わない
- 拡大倍率の実行時設定・動的変更（固定値 1.2 のみ）

## Decisions

### Y縮小と同一のchargeAmount変数を再利用する

`Player.ts` は既に `chargeAmount`（0〜1）を持ち、毎フレーム `scaleY = 1.0 - 0.2 * chargeAmount` を計算している。X方向は同じ変数で `scaleX = 1.0 + 0.2 * chargeAmount` を計算する。

**Alternatives considered:**  
独立したチャージ値を使う案 → 同じポインタを使えば実装が1行で済むため却下。

### 倍率を定数として定義する

`Player.ts` に `CHARGE_SCALE_X_MAX = 0.2`（増加量）として定数化し、後から値を調整しやすくする。

## Risks / Trade-offs

- [視覚的違和感] 横幅が物理的な当たり判定より広く見えることで、障害物に当たっていないのに当たったように感じる可能性 → 倍率を控えめ（1.2倍）にすることで軽減。動作確認後に調整する。
