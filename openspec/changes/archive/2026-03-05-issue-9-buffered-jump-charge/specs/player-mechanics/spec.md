# Spec Delta: player-mechanics

## MODIFIED Requirements

### Requirement: チャージジャンプ入力

プレーヤーはスペースキー（PC）またはタップ（スマホ）の長押しでジャンプパワーをチャージし、リリース時にジャンプが発動する。チャージ量は押下時間に比例し、最大500msで上限（100%）に達する。上限到達後は押し続けてもチャージ量は変化しない。空中での入力はバッファリングされ、着地時にチャージが自動開始される。

#### Scenario: 短押しで小ジャンプ

- **WHEN** プレーヤーが200ms未満で入力をリリースする
- **THEN** 初速400px/s未満の小ジャンプが発動する

#### Scenario: 長押しで大ジャンプ

- **WHEN** プレーヤーが500ms以上押し続けてからリリースする
- **THEN** 初速900px/sの最大ジャンプが発動する

#### Scenario: 上限到達後の待機

- **WHEN** プレーヤーが500ms以上押し続けた状態で待機する
- **THEN** チャージ量は100%のまま維持され、リリースするまでジャンプは発動しない

#### Scenario: 空中でのジャンプ入力バッファリング

- **WHEN** プレーヤーが空中にいる状態で入力を開始する（タッチダウンまたはスペースキーダウン）
- **THEN** ジャンプは即時発動しない
- **AND** 「着地時にチャージを開始する」バッファフラグが立てられる

#### Scenario: 空中でのリリースによるバッファキャンセル

- **WHEN** プレーヤーが空中にいる状態で入力を開始した後、空中のままリリースする
- **THEN** バッファフラグがキャンセルされる
- **AND** 着地後もチャージは開始されない

#### Scenario: バッファリング後の着地によるチャージ開始

- **WHEN** 空中でバッファフラグが立った状態でプレーヤーが着地する
- **THEN** 着地した瞬間をチャージ開始時刻として記録する
- **AND** チャージ量は着地時刻からのカウントで計算される（空中で押していた時間は含まない）
- **AND** バッファフラグはクリアされる

#### Scenario: バッファリング後のリリースによるジャンプ発動

- **WHEN** バッファからチャージが開始された後にプレーヤーがリリースする
- **THEN** 「着地時刻からリリースまでの時間」に応じたチャージ量でジャンプが発動する
