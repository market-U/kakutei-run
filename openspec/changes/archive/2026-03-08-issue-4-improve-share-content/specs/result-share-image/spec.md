# Spec: result-share-image

## ADDED Requirements

### Requirement: 感動スコアによるシェアコメント選出

`CommentManager` はゲームプレイ中に流れたコメントの中から 1 件をシェア用候補として保持しなければならない（SHALL）。候補はコメントがスポーンされるたびに感動スコア（0〜100 のランダム整数）で評価し、保存済みスコアより高い場合のみ置き換える。

#### Scenario: ゲーム開始時に候補をリセットする

- **WHEN** `startGame()` が呼ばれる
- **THEN** シェア用コメント候補が `null` にリセットされる

#### Scenario: スポーン時に感動スコアで評価する

- **WHEN** コメントがスポーンされる
- **THEN** 0〜100 のランダムな感動スコアが生成される
- **AND** 保存済みスコアより高い場合、候補コメントとスコアが置き換えられる
- **AND** 保存済みスコア以下の場合、候補は変更されない

#### Scenario: シェア用候補を取得する

- **WHEN** `getShareComment()` が呼ばれる
- **THEN** 保存中のコメント文字列を返す
- **AND** 候補が存在しない場合は `null` を返す

---

### Requirement: Canvas API によるシェア用画像生成

リザルト画面のシェアボタン押下時に Canvas API で 1080×1080px の画像を生成しなければならない（SHALL）。

#### Scenario: クリア時の画像生成

- **WHEN** クリア結果でシェアボタンが押される
- **THEN** `public/assets/ui/share_clear.png` をベースに Canvas へ描画する
- **AND** 難易度名を上部に重ねて描画する
- **AND** レシート取得率・シェア用コメント・バージョンを下部に重ねて描画する

#### Scenario: ゲームオーバー時の画像生成

- **WHEN** ゲームオーバー結果でシェアボタンが押される
- **THEN** `public/assets/ui/share_gameover.png` をベースに Canvas へ描画する
- **AND** 難易度名を上部に重ねて描画する
- **AND** レシート取得率・シェア用コメント・バージョンを下部に重ねて描画する

#### Scenario: シェア用コメントが存在しない場合

- **WHEN** シェアボタンが押された時点でシェア用コメント候補が `null` である
- **THEN** コメント欄を空白として画像を生成する
