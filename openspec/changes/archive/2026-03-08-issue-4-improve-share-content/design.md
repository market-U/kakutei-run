# Design: リザルトシェア内容の改善

## Context

現在のシェア機能は `src/ui/ResultUI.ts` の `doShare()` に実装されている。シェアテキストはベタ書きで、Web Share API 対応時は `navigator.share({ text })` を、非対応時は Twitter intent URL を開く。画像共有は未対応。

コメントデータは `window.__commentsData` にキャッシュされており、`ResultUI.ts` から直接参照できる。Vite の `define` で `__APP_VERSION__` がビルド時に埋め込まれている。

## Goals / Non-Goals

**Goals:**

- シェアテキストを結果・難易度・スコア・バージョン・URL・ハッシュタグを含む形式に改善する
- Canvas API で 1080×1080px のシェア用画像を生成し、Web Share API の `files` オプションで画像ごと共有する
- 画像が SNS に届いても単体で結果が伝わるよう、難易度・スコア・コメント・バージョンをオーバーレイする
- 外部ライブラリを追加しない

**Non-Goals:**

- サーバーサイドでの動的 OGP 画像生成
- スクリーンショット（html2canvas 等）による画面キャプチャ
- X（Twitter）への画像直接添付（intent URL 経由では不可能なため非対応）

## Decisions

### 画像生成に Canvas API を使用する

`html2canvas` は CSS の再現に癖があり保守コストが高い。Canvas API は標準仕様でありライブラリ追加不要。シェア用に特化したカード画像を描画するため、Canvas API で十分かつ最適。

### シェア画像は 1080×1080px（正方形）

X・Bluesky・Instagram など主要 SNS で安定して表示される。縦長（ストーリー型）はプラットフォーム差異が大きいため不採用。横長（OGP 型）はモバイルのシェアシートでトリミングされることがあるため不採用。

### ベース画像を 2 枚用意する（クリア / ゲームオーバー）

結果の雰囲気を視覚的に伝えるためにベース画像を分ける。Canvas でのテキストオーバーレイにより難易度・スコア・コメントは動的に生成する。

### テキストオーバーレイの配置

- 難易度名: 上寄せ（画像内のビジュアルを遮らない位置）
- スコア・ランダムコメント・バージョン: 下寄せ（まとめて配置）

### シェア用コメントは「感動スコア」方式で選出する

`CommentManager` にコメント候補オブジェクト `{ comment: string, score: number }` を 1 つ保持する。コメントがスポーンされるたびに 0〜100 のランダムな感動スコアを生成し、保存済みのスコアより高ければ候補を置き換える。ゲーム終了時に残った候補コメントをリザルトシェアに使用する。

この方式はリザーバーサンプリング（k=1）と統計的にほぼ同等で、実際に流れたコメントから均等に 1 件を選出できる。メモリは O(1) でスポーン時の処理コストも `Math.random()` 1 回の比較のみ。将来的にリザルト画面に感動スコアを表示する拡張も容易。

### Web Share API 非対応時は Twitter URL（テキストのみ）にフォールバック

`navigator.canShare({ files: [...] })` で画像付き共有の可否を判定し、非対応の場合はテキストのみの Twitter intent URL を開く。PC ブラウザはテキストシェアのみとなる。

### 本文テキストは URL + ハッシュタグのみ

iOS Bluesky アプリ等、画像＋本文を同時に受け取れない SNS クライアントが存在する。画像に難易度・スコア等を含めることで本文が消えても情報が欠落しない設計とする。

## Risks / Trade-offs

- **[リスク] Web Share API の `files` 対応状況がブラウザ・OS 依存** → `navigator.canShare()` で事前チェックし、非対応時はテキストのみフォールバックする
- **[リスク] Canvas でのフォント描画が環境によって異なる** → システムフォントに依存する部分が生じるが、ゲームの雰囲気に影響するほどではない。許容範囲とする
- **[トレードオフ] 本文テキストが消えると URL が届かない** → 画像にバージョンを入れることで代替。URL はおまけ扱い

## Migration Plan

変更対象は `src/ui/ResultUI.ts`（`doShare()` 改修）と `src/systems/CommentManager.ts`（感動スコア候補の追加）の 2 ファイルのみ。既存の SNS シェアフロー（ボタン→Web Share API / Twitter URL）は維持するため、ロールバックはファイル差し戻しで完結する。
