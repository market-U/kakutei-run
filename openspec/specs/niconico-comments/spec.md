# Spec: niconico-comments

## Requirement: コメントデータの読み込み

システムはタイトル画面の表示時に `VITE_COMMENTS_URL` 環境変数で指定された Azure Blob Storage のパブリック URL から `comments.json` を fetch し、コメントデータを読み込まなければならない（SHALL）。
fetch は 1 回あたり 5 秒のタイムアウトを設け、失敗時は最大 3 回リトライしなければならない（SHALL）。
3 回すべて失敗した場合はバンドル済みの `/comments.json` を fetch し、それも失敗した場合はハードコードされたフォールバックコメントを使用しなければならない（SHALL）。
fetch が完了するまで（成功・フォールバック問わず）難易度選択ボタンは無効化され、ローディングスピナーを表示しなければならない（SHALL）。

### Scenario: Blob URL から正常に fetch できる場合

- **WHEN** タイトル画面が表示される
- **THEN** `VITE_COMMENTS_URL` の URL から JSON を取得し、コメントデータとして保持する
- **AND** ローディングスピナーが非表示になり、難易度選択ボタンが有効化される

### Scenario: fetch がタイムアウトする場合

- **WHEN** `VITE_COMMENTS_URL` への fetch が 5 秒以内に完了しない
- **THEN** そのリクエストをキャンセルし、リトライカウントを 1 増やして再試行する

### Scenario: 3 回すべてのリトライが失敗した場合

- **WHEN** `VITE_COMMENTS_URL` への fetch が 3 回すべて失敗する（タイムアウト・エラー問わず）
- **THEN** バンドル済みの `/comments.json` を fetch してフォールバックとして使用する
- **AND** コンソールに警告を出力する

### Scenario: バンドル済み JSON のフォールバックも失敗した場合

- **WHEN** バンドル済み `/comments.json` の fetch も失敗する
- **THEN** ハードコードされたフォールバックコメント定数を使用する

### Scenario: fetch 中のローディング表示

- **WHEN** タイトル画面が表示され fetch が進行中である
- **THEN** DaisyUI の `loading-spinner` が難易度ボタン領域に表示される
- **AND** 難易度選択ボタンは disabled 状態である

### Scenario: `VITE_COMMENTS_URL` が未設定の場合

- **WHEN** `VITE_COMMENTS_URL` が空文字または未定義である
- **THEN** Blob fetch をスキップしてバンドル済み `/comments.json` を直接 fetch する

### Scenario: fetch に失敗した場合

- **WHEN** すべての fetch 試行が失敗する
- **THEN** フォールバックのハードコードコメント配列を使用し、エラーは握り潰さずコンソールへ出力する

## Requirement: コメントのカテゴリ分類

コメントデータは以下のカテゴリに分類されなければならない（SHALL）。

- `common`：常時アクティブプールに含まれる共通コメント
- `difficulty.easy / normal / hard`：難易度別コメント
- `events.start`：ゲーム開始時のコメント
- `events.stumble`：転倒イベント時のコメント
- `events.backPain`：腰痛（魔女被弾）イベント時のコメント
- `events.goal`：ゴール到達イベント時のコメント

### Scenario: ゲーム開始時のアクティブプール合成

- **WHEN** ゲームが開始され難易度が決定される
- **THEN** `common` と対応する `difficulty[id]` を合成したアクティブプールが生成される

### Scenario: イベント発生時のバーストコメント

- **WHEN** ゲームイベント（開始・転倒・腰痛・ゴール）が発生する
- **THEN** 対応する `events[type]` のプールのみからコメントが選ばれてバースト投入される

---

## Requirement: コメントの表示形式

コメントは Phaser キャンバスの前面（depth: 20）に DotGothic16 フォントで表示されなければならない（SHALL）。
スタイルは白文字・黒ドロップシャドウ・半透過（alpha: 0.82）とする。

### Scenario: コメントの描画スタイル

- **WHEN** コメントオブジェクトが生成される
- **THEN** フォントは DotGothic16・サイズ 48px・白・ドロップシャドウ（offset: 1, blur: 3, 黒）・alpha: 0.82 で描画される

### Scenario: ゲームオブジェクトの前面に表示

- **WHEN** コメントが画面に表示される
- **THEN** depth: 20 で描画され、ゲームオブジェクト（地面: 2, ゴール: 5）の前面かつクリア演出（29-30）の背面に配置される

---

## Requirement: コメントのスクロール速度

すべてのコメントは同一の画面横断時間（`CROSSING_DURATION`）で画面を横断しなければならない（SHALL）。
これにより文字数が長いコメントほど速く流れる。

### Scenario: 短いコメントの速度

- **WHEN** 「草」などの短いコメントが生成される
- **THEN** `(CANVAS_W + text.width) / CROSSING_DURATION` で計算された低速で流れる

### Scenario: 長いコメントの速度

- **WHEN** 「確定申告やばすぎて草不可避ｗｗ」などの長いコメントが生成される
- **THEN** `(CANVAS_W + text.width) / CROSSING_DURATION` で計算された高速で流れる

### Scenario: ゲームスクロール速度との独立性

- **WHEN** ゲームの地面スクロール速度が変化する（腰痛スロー等）
- **THEN** コメントの流れる速度は影響を受けない

---

## Requirement: レーン管理による重なり防止

コメントは複数のレーンに分散して表示され、同一レーン内でのコメント重なりを防がなければならない（SHALL）。

### Scenario: レーン選択

- **WHEN** 新しいコメントをスポーンする
- **THEN** 前のコメントの末尾が画面右端から 30% 以上進んだレーン（最もゆとりのあるレーン）が選択される

### Scenario: 全レーンが混雑している場合

- **WHEN** すべてのレーンで前コメントが 30% 未満しか進んでいない
- **THEN** そのスポーンはスキップされる（コメントは流さない）

---

## Requirement: イベント時のバースト投入

ゲームイベント発生時はコメントを高密度で投入しなければならない（SHALL）。

| イベント | バースト数 |
| -------- | ---------- |
| 開始     | 4 本       |
| 転倒     | 6 本       |
| 腰痛     | 4 本       |
| ゴール   | 9 本       |

### Scenario: 転倒時のバースト

- **WHEN** `stoneHit` イベントが発火する
- **THEN** ランダムな間隔（200〜600ms）で 6 本のコメントが投入される

### Scenario: ゴール時のバースト（ループ）

- **WHEN** ゴール到達処理が発火する
- **THEN** ゴール系コメントが 9 本ずつ無限ループでバースト投入され続ける

### Scenario: ゲームオーバー時のバースト（ループ）

- **WHEN** 敵がプレイヤーに到達してゲームオーバーになる
- **THEN** 転倒系コメントが 6 本ずつ無限ループでバースト投入され続ける

### Scenario: 新イベントによるバーストキャンセル

- **WHEN** バースト投入中に新たなイベントが発生する
- **THEN** 前イベントのバーストキューがクリアされ、新イベントのバーストが開始される

---

## Requirement: コメント表示の ON/OFF

プレイヤーはゲームプレイ中にコメント表示をトグルで切り替えられなければならない（SHALL）。

### Scenario: コメントを OFF にする

- **WHEN** コメントトグルボタンを押して OFF にする
- **THEN** 画面上のコメントが即座に非表示になり、新規スポーンが停止する

### Scenario: コメントを ON にする

- **WHEN** コメントトグルボタンを押して ON にする
- **THEN** 新規コメントのスポーンが再開される

### Scenario: トグル状態の視覚フィードバック

- **WHEN** コメントが OFF 状態である
- **THEN** トグルボタンの外観が OFF 状態であることを示す（例: 半透過）
